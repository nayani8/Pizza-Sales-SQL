# 🍕 Pizza Sales Analysis using SQL

## 📌 Project Overview

This project focuses on analyzing pizza sales data using SQL to extract meaningful business insights. It demonstrates the use of SQL queries ranging from basic data retrieval to advanced analytical operations for solving real-world business problems.

The analysis includes sales trends, revenue generation, customer ordering patterns, and product performance using relational database concepts.

---

## 🎯 Objectives

* Analyze overall pizza sales performance.
* Calculate total revenue generated.
* Identify the most popular pizzas.
* Analyze sales by pizza category and size.
* Determine revenue contribution by each category.
* Track cumulative revenue over time.
* Rank top-performing pizzas within each category.

---

## 🛠️ Technologies Used

* MySQL
* SQL
* Relational Database

---

## 📂 Dataset

The project uses four relational datasets:

* **orders.csv** – Customer order information
* **order_details.csv** – Quantity of pizzas ordered
* **pizzas.csv** – Pizza sizes and prices
* **pizza_types.csv** – Pizza names, ingredients, and categories

## 📈 Business Questions Solved

### Basic Analysis

* Retrieve the total number of orders.
* Calculate total revenue generated.
* Identify the highest-priced pizza.
* Find the most commonly ordered pizza size.
* List the top 5 most ordered pizzas.

### Intermediate Analysis

* Calculate total quantity sold by category.
* Analyze hourly order distribution.
* Find the average number of pizzas ordered per day.
* Identify the top revenue-generating pizzas.

### Advanced Analysis

* Calculate each pizza category's contribution to total revenue.
* Compute cumulative revenue over time.
* Rank the highest-selling pizzas within each category.
